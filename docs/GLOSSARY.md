# Глоссарий (Справочник терминов)

Единый справочник терминов проекта Finpi OCR. Обеспечивает однозначность и отсутствие разночтений.

---

## Архитектура

| Термин | Определение |
|--------|-------------|
| **Домен (Domain)** | Независимый модуль системы с чётким ЦКП. D1 (Extraction), D2 (Parsing), D3 (Categorization). |
| **Pipeline** | Последовательность этапов обработки данных. |
| **Phase** | Крупный этап пайплайна (Pre-OCR, OCR, Post-OCR). Содержит Stages. |
| **Stage** | Этап внутри Phase. Содержит Operations. Имеет свой ЦКП. |
| **Operation** | Атомарная операция внутри Stage. Решает одну проблему (SRP). Имеет свой ЦКП. |

---

## Принципы

| Термин | Определение |
|--------|-------------|
| **ЦКП** | Ценный Конечный Продукт. Измеримый результат работы Stage/Operation. |
| **SRP** | Single Responsibility Principle. Один компонент решает одну проблему. |
| **Контракт** | Формальное описание Input/Output компонента. Гарантия совместимости. |
| **Ground Truth** | Эталонные данные, созданные ЧЕЛОВЕКОМ вручную. 100% точность. D1 НЕ трогает. Используется для тестирования. |

---

## Подходы

| Термин | Определение |
|--------|-------------|
| **Системное решение** | Решение на уровне паттерна, работает для класса случаев. |
| **Костыль (Хардкод)** | Решение для одного конкретного случая. ЗАПРЕЩЕНО. |
| **Конфиг** | Параметры в `config/settings.py`. Системный подход к настройке. |

---

## Иерархия (визуально)

```
Домен (D1)
  └── Pipeline
        └── Phase (Pre-OCR)
              └── Stage (Pre-OCR Pipeline)
                    ├── Operation (ImageCompressor)
                    └── Operation (Grayscale)
```
